# 🔧 دليل الإصلاح الطارئ - Beiruttime OSINT Pro

## المشكلة
ظهر خطأ حرج في الموقع وتوقفت جميع الصفحات عن العمل (سجل الأخبار، بنوك المعلومات، لوحة المؤشرات، التقارير التنفيذية).

## السبب المحتمل
1. بيانات فاسدة في قاعدة البيانات (ترميز مزدوج للنصوص العربية)
2. جداول قاعدة البيانات مفقودة أو تالفة
3. ملفات JavaScript/CSS لا يتم تحميلها بشكل صحيح
4. خيارات WordPress (Options) تحتوي على قيم مكررة أو فاسدة

## خطوات الإصلاح

### الطريقة 1: تشغيل سكريبت الإصلاح التلقائي (موصى به)

1. ارفع ملف `fix-so-core.php` إلى المجلد:
   ```
   /wp-content/plugins/beiruttime-osint-pro-v17.4.2-context-intent-fix/
   ```

2. افتح المتصفح وانتقل إلى:
   ```
   https://yoursite.com/wp-content/plugins/beiruttime-osint-pro-v17.4.2-context-intent-fix/fix-so-core.php
   ```

3. سيقوم السكريبت تلقائياً بـ:
   - إنشاء/إصلاح جداول قاعدة البيانات
   - تنظيف البيانات الفاسدة (الترميز المزدوج)
   - إزالة التكرارات من بنوك المعلومات
   - إعادة تعيين القيم الافتراضية
   - مسح ذاكرة التخزين المؤقت

4. **مهم:** احذف ملف `fix-so-core.php` بعد الاستخدام لأسباب أمنية!

### الطريقة 2: الإصلاح اليدوي عبر قاعدة البيانات

إذا لم تستطع تشغيل السكريبت، نفذ هذه الاستعلامات في phpMyAdmin:

```sql
-- 1. إصلاح الجداول
OPTIMIZE TABLE wp_so_news_events;
OPTIMIZE TABLE wp_so_manual_learning;
OPTIMIZE TABLE wp_so_sent_alerts;

-- 2. حذف البيانات المكررة من بنوك التعلم
DELETE FROM wp_options WHERE option_name LIKE 'sod_bank_%' AND option_value = '';

-- 3. تفريغ ذاكرة التخزين المؤقت
DELETE FROM wp_options WHERE option_name LIKE '_transient_sod_%';
```

### الطريقة 3: تعطيل الإضافة وإعادة تفعيلها

1. عبر FTP أو File Manager:
   - أعد تسمية مجلد الإضافة مؤقتاً:
     ```
     beiruttime-osint-pro-v17.4.2-context-intent-fixDISABLED
     ```

2. انتقل إلى لوحة تحكم ووردبريس > الإضافات

3. أعد تسمية المجلد إلى اسمه الأصلي

4. فعّل الإضافة مرة أخرى (سيتم إنشاء الجداول تلقائياً)

## بعد الإصلاح

1. امسح ذاكرة التخزين المؤقت للمتصفح (Ctrl+F5)
2. انتقل إلى صفحة "سجل الأخبار" وتحقق من ظهور البيانات
3. انتقل إلى "بنوك المعلومات" وتحقق من عدم وجود نصوص مشفرة
4. اختبر "لوحة المؤشرات" و"التقارير التنفيذية"

## إذا استمرت المشكلة

1. فعّل وضع التصحيح في ووردبريس بإضافة هذا السطر إلى `wp-config.php`:
   ```php
   define('WP_DEBUG', true);
   define('WP_DEBUG_LOG', true);
   ```

2. تحقق من ملف `wp-content/debug.log` لمعرفة الخطأ الدقيق

3. راجع سجل أخطاء PHP في الخادم (error_log)

## الملفات المهمة للإضافة

- الملف الرئيسي: `beiruttime-osint-pro.php`
- سكريبت الإصلاح: `fix-so-core.php`
- ملفات JS: `/assets/js/newslog-admin.js`, `/assets/js/db-admin.js`
- ملفات CSS: `/assets/css/admin-pages.css`

## الوقاية مستقبلاً

1. احتفظ بنسخة احتياطية منتظمة من قاعدة البيانات
2. لا تقم بتعديل البيانات مباشرة في قاعدة البيانات إلا إذا كنت خبيراً
3. حدّث الإضافة فقط بعد أخذ نسخة احتياطية
4. استخدم بيئات اختبار قبل التطبيق على الموقع الحي
